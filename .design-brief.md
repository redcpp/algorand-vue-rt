> Working document for the redesign swarm. **Read this before touching any code.**
> Branch: `redesign/stochastic-ledger`. Preview deploy target: `algo-vue-rt-preview.surge.sh`. Do not touch master.

## The concept

A live-data art piece framed as a **broadside from an alternate-timeline financial magazine** — one where blockchain went editorial instead of corporate. The page reads like a single-issue printed page, not an "app". The Algorand TestNet is the live ticker that drives the typography.

**The one thing people remember:** every falling block is a small printed artifact — a hand-stamped rectangle of ink with its round number printed on top, in a curated five-ink palette, with a soft edge bleed. The canvas becomes a printing press, not a screen.

## Tone

- Editorial / broadside / letterpress.
- Restrained but confident. **Refined, not maximalist.**
- Weight comes from typographic scale and color discipline — not effects.
- Reverence for the data. The round number is the masthead's heartbeat.

## Color tokens (locked — use exactly these)

```css
:root {
  /* paper & ink */
  --paper:         #F5F1E8;   /* warm off-white, aged broadside */
  --paper-deep:    #ECE6D6;   /* secondary panel ground */
  --ink:           #0A0A0A;   /* sumi black, primary text */
  --ink-soft:      #2A2520;   /* secondary text */
  --rule:          #C9C2B0;   /* hairline rules */
  --taupe:         #A8A29C;   /* tertiary text / footnote */

  /* the five inks (block palette + accents) */
  --vermillion:    #D63B1F;
  --forest:        #1B4D3E;
  --ochre:         #C99B3B;
  --indigo:        #2B3A67;
  --sumi:          #0A0A0A;   /* same as ink, reused for block tone */

  /* atmospherics */
  --paper-shadow:  rgba(10, 10, 10, 0.08);
  --grain-opacity: 0.06;
}
```

**Do NOT introduce other colors.** The whole composition lives on these tokens.

**Block colors when stamping new blocks**: random pick from `[vermillion, forest, ochre, indigo, sumi]` — never the old random-RGB.

## Typography

Load via Google Fonts in `<head>` (preconnect + preload):

- **Fraunces** — variable serif, with `opsz` (optical sizing) and `SOFT` axes. Use weight 900 + opsz 144 for the masthead, weight 700 + opsz 72 for the giant round number, weight 400 + opsz 14 for editorial body.
- **JetBrains Mono** — weights 400 + 500. Used for all numeric data, timestamps, footnote citations, captions under the canvas.

Font fallbacks: `Fraunces, 'Times New Roman', Georgia, serif` and `'JetBrains Mono', 'SF Mono', Menlo, monospace`.

## The giant round number (left rail)

- One digit per line.
- Font: Fraunces, weight 700, opsz 144, letter-spacing -0.03em.
- Color: `--ink`.
- Tracks the live last round in real-time. When the round changes, each digit that differs from the previous round briefly transitions (a 600ms fade through `--vermillion` back to `--ink`) — like ink pressing.
- The label "ROUND" above it: Fraunces weight 400 italic, opsz 24, vertical (writing-mode), centered.

## The canvas (the printing press)

- Frame: 1px hairline `--rule` border with 24px padding inside, paper-deep background tint.
- Width: ~720px desktop, full container width mobile. 4:3 aspect ratio.
- p5 setup:
  - `background('#ECE6D6')` (paper-deep) on setup; never wipe — block trails persist like ink prints.
  - On each new round, stamp a rectangle:
    - Color: random from the 5 inks.
    - Width: 96–160px, height: 48–72px (varied per stamp).
    - Position: random within the canvas with margin from edges.
    - Rotation: ±4° random.
    - Edge treatment: rectangle drawn with `noiseDetail` jitter or a subtle stroke; then a slightly darker outline 1px below-right (suggests pressure/bleed).
    - Overlay: print the round number on top of the block in JetBrains Mono, color `--paper`, opacity 0.85, small (12–14px). Center-aligned on the block.
  - Old stamps don't move (no falling). They stay where they were printed. Newer stamps overlay older ones — the canvas becomes a denser print over time.
  - Throttle/clear: after ~40 stamps, fade the oldest stamps subtly by drawing a low-alpha `--paper-deep` rectangle over the whole canvas every 5 stamps. This gives the "the press wipes occasionally" effect without ever fully clearing.

**This is a major behavioral change from the old "falling" model.** The user has approved it. The new metaphor: the canvas is a printing press accumulating impressions, not a Tetris well.

## Decorative details (the surprises)

- **Page grain**: an inline SVG noise overlay fixed over the whole viewport, opacity `--grain-opacity`, mix-blend-mode: multiply. Use SVG `<filter><feTurbulence>` (no external assets).
- **Masthead ornaments**: tiny `§ · § · §` glyphs between sections, in `--taupe`, Fraunces italic.
- **Custom cursor over canvas**: a small SVG crosshair + the next round number tracking near it (data-attr updated on round change). Use `cursor: url(...) x y, crosshair;` with a 16×16 SVG.
- **Stop-press indicator**: a `◆` glyph in `--vermillion` that pulses (1.2s ease-in-out infinite) for 800ms after each new round, then settles. CSS animation triggered by adding/removing a class.
- **Hairline rules**: 1px lines in `--rule` separating masthead from body and body from footnotes.
- **Footnotes**: JetBrains Mono 11px, `--taupe`, numbered with superscript Unicode (¹ ² ³).
- **Type set in metal**: optional small decorative element — a 3-character monogram "AVR" (Algorand · Vue · Real-time) tucked in a corner of the footer in `--vermillion`, Fraunces 900 opsz 24.

## Motion

Restrained. Three motion moments:
1. **Page load**: masthead text fades up (300ms ease-out), giant round number scales from 0.96→1.0 + fades in (500ms ease-out), canvas frame draws in (border-image expand 400ms), footnotes fade in last (200ms delay 600ms). Stagger total ~900ms.
2. **New round arrives**: the digit(s) that changed fade through vermillion (600ms), stop-press indicator pulses, an audible-looking "ink press" — the canvas frame briefly nudges down 1px and back (80ms each way).
3. **Grain**: the SVG noise can have a slow `animateTransform` that shifts it 4px on a 6s loop, so the page feels alive without distracting.

**Do not** add hover micro-interactions to every element. The piece is contemplative.

## What stays the same

- The data layer in `src/components/Algorand.vue`: the polling loop calling AlgoNode and emitting `lastRound` events. **Do not break this**. You may rename the emitted event or add a second event for "round-changed-with-diff" if helpful, but the underlying logic stays.
- `algosdk v2` + AlgoNode endpoint.
- The build system (Vue CLI 4, the `cross-env NODE_OPTIONS=--openssl-legacy-provider` workaround) — do not migrate to Vite in this redesign.

## What changes

- `public/index.html`: title, fonts preload, grain SVG (or a `<style>` for grain), favicon (a small `§` glyph as SVG favicon would be perfect), meta description.
- `src/App.vue`: complete layout rewrite — masthead, two-column grid, footnotes footer.
- `src/components/AlgoViz.vue`: complete rewrite — the printing press p5 sketch (stamps, not falling blocks), the canvas frame, the caption + stop-press indicator.
- `src/components/Algorand.vue`: minor — emit the round value and maybe a "diff array" of which digits changed since last emit (helps the giant number animation).
- New file: `src/assets/grain.svg` (optional — inline SVG in App.vue also fine).
- New file: `src/styles/tokens.css` for the color/type tokens, imported once from `main.js`.

## What does NOT exist in this redesign

- No random RGB.
- No "falling" blocks (the metaphor is now printing impressions).
- No gradient anywhere on the page.
- No emojis in the UI.
- No box-shadows with blur radius >2px (this is print culture — no glassmorphism).
- No purple. Especially no purple.
